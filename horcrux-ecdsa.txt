			Запуск horcrux c ECDSA форматом подписи.
Есть 3 папки - cosigner_1, cosigner_2, cosigner_3(создание: mkdir cosigner_1 cosigner_2 cosigner_3)

1. Настройка конфигов.
	1.1 Создание приватных частей для каждой ноды.
	
	    ./horcrux create-shares --ecdsa - 2 3
	    create-shares - команда для создания приватных частей
	    --ecdsa - указывается для создания приватных частей эфировского фармата, для heimall
	    - прочерк, без ecdsa это необходимо было для указания файла с приватным ключом, который нужно разбить на части, в нашем случае нет разбиения, "распределенный" приватный ключ формируется заново.
	    2 - порог для подписи(2 из 3). Т.е. кластер может выдержать 1 отказ из 3.
	    3 - кол-во участников подписи, т.е. нод
	    
	    вывод: для кажной ноды формируются 2 файла - cmp_config_1.txt и cmp_presig_1.txt, 
	    нужно в папку для каждого подписывающего переместить эти файлы и убрать номер, т.е.
	    cmp_config.txt, cmp_presig.txt в каждой папке.
	    
	    переместить все по папкам подписывающих:
 		mv cmp_config_1.txt cosigner_1/cmp_config.txt
 		mv cmp_config_2.txt cosigner_2/cmp_config.txt
 		mv cmp_config_3.txt cosigner_3/cmp_config.txt
 		mv cmp_presig_1.txt cosigner_1/cmp_presig.txt
 		mv cmp_presig_2.txt cosigner_2/cmp_presig.txt
 		mv cmp_presig_3.txt cosigner_3/cmp_presig.txt
 		mv private_share_1.json cosigner_1/share.json
 		mv private_share_2.json cosigner_2/share.json
 		mv private_share_3.json cosigner_3/share.json
	    
	    ./horcrux cosigner address --ecdsa --home cosigner_1
	    cosigner address - команда для вывода адреса из любой приватной части, можно посмотреть адрес в формате эфира и публичный ключ. У каждой ноды они будут очевидно одинаковы, адрес и публичный ключ один, а приватный "распределен" по кластеру.
	    --ecdsa - подпись эфировская
	    --home папка ноды с приватными файлами и конфигом
	    
	  
	1.2 Создание конфигов нод. 
	    Команды для 3 нод:
	     ./horcrux config init --home cosigner_1 chain-1 "tcp://localhost:8181" -c -p "tcp://localhost:8192|2,tcp://localhost:8193|3" -l "tcp://localhost:8191" -t 2 --timeout 1500ms

./horcrux config init --home cosigner_2 chain-1 "tcp://localhost:8182" -c -p "tcp://localhost:8191|1,tcp://localhost:8193|3" -l "tcp://localhost:8192" -t 2 --timeout 1500ms

./horcrux config init --home cosigner_3 chain-1  "tcp://localhost:8183" -c -p "tcp://localhost:8191|1,tcp://localhost:8192|2" -l "tcp://localhost:8193" -t 2 --timeout 1500ms

	     config init - команда для создания конфига
	     --home - указываем куда нужно положить файл конфига, папка каждого подписывающего
	     chain-1 - номер сети, цепочки, mainnet testnet и т.д.
	     "tcp://localhost:8181"  - это адрес по которому tendermint или heimdall будут 
	     общаться с этой нодой
	     -с - cosigner полный флаг, означает что нода это подписывающий, т.е. кластерный 
	     режим, можно сделать single, когда одна нода подписывает данные
	     -p - peers полный флаг, указываем адреса других нод и их номера
	     -l - listen полный флаг, указываем на каком адресе мы будем находится
	     -t - threshold полный флаг, указываем порог для подписи
	     --timeout - указываем таймаут ожидания ответа от других нод

2. Запуск кластера
	./horcrux cosigner start --ecdsa --home cosigner_1
	cosigner start - команда запуска ноды
	--ecdsa - подпись эфировская
	--home папка ноды с приватными файлами и конфигом
